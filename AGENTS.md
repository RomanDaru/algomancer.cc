# Agent Workflow

- Before closing any task that changes code, config, tests, docs, or runtime behavior, append a short entry to `docs/CHANGELOG.md`.
- Keep changelog entries newest-first within the current date section.
- Each entry should include:
  - branch name
  - short summary of the change
  - key areas/files touched
  - verification performed
- Do not delete historical entries unless the user explicitly asks for changelog cleanup.
